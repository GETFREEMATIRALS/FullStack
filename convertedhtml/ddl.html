Data Definition on language
Date Definition on language ( DDL) is used to create, Rename , drop the columns or tables. If we want to create a table we need to learn
about two things mainly :
1. DATA TYPES : It decides which type of data has to be stored in columns. They are 3 types of data types :
Number
()
Whenever we are using number data type as
Number, it is always numerical in nature.
char ()
whenever we are se ng the datatype as character, it
should be always Alphabetical in nature
varchar
()
It is present in the old version of the database, and
for the latest version. We are using varchar2()
function, and it accepts both numerical and
alphabetical values.
2. CONSTRAINTS: Constraints are the conditions the columns must flow while taking the records as input. They are two main constraints:
UNIQUE
KEY
It doesn't allow duplicate values, but it allows null
values
NOT NULL
KEY
It doesn't allow null values, but it allows duplicate
values
Syntax for Creating a table:
SQL> create table table name
(Column name1 datatype(size) Constrains,
Column name2 datatype(size) Constrains,
Column name3 datatype(size) Constrains..................);
NOTE: Here 'SIZE' represents a number of characters.
EX: number (3) => it allows numbers from 1 to 999 only.
char (4) => it allows alphabets up to 4 characters.
SQL> create table std
( sno number (5) unique sname char (5) not null course char (5) not null );
OUTPUT:
Table created.
