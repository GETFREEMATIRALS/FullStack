Transaction Control language:
Transactions group a set of tasks into a single execution unit. Each transaction begins with a specific task and ends when all the tasks in the group
successfully complete.
Uses of TCL commands:
1. To make DML commands permanent
2. To retrieve deleted records (Other than truncated records)
3. To delete the tables from recycle bin
TCL Commands:
COMMIT:
All DML commands are temporary changes and to make them permanent changes we have to use TCL command called ‘COMMIT’.
Syntax for Commit command:
SQL>Commit;
NOTE: we have to use this command for inserting the records into table
SQL> select * from cat;
TABLE_NAME TABLE_TYPE
------------------------------ -----------
DEPT TABLE
EMP TABLE
BONUS TABLE
SALGRADE TABLE 10 rows selected.
SQL> create table clg(
2 sno number(10) unique,
3 sname char(10) not null,
4 course char(10)not null);
Table created.
SQL> insert all into clg values( 1 , 'AJAY' , 'SQL')
2 into clg values(2 , 'RAJA' , 'JAVA' )
3 into clg values(3 , 'RANI' , 'MANUVAL' ) select * from dual;
3 rows created.
/* using TCL command to make these records permanent*/
SQL> commit;
Commit complete.
SQL> select * from clg;

SAVEPOINT: It acts break point between Commit command and Rollback command
Syntax for Savepoint command:
SQL>Savepoint (Savepoint_Name);
/* inserting 3 more records into clg table*/
SQL> insert all into clg values( 4 , 'MACHA' , 'SQL')
2 into clg values(5 , 'MACHI' , 'WEB' )
3 into clg values(6 , 'KAJA' , 'PYTHON' ) select * from dual;
3 rows created.
/* creating save point*/
SQL> savepoint A;
Savepoint created.
/* Displaying clg records*/
SQL> select * from clg;

---------- ---------- ----------
1. AJAY SQL
2. RAJA JAVA
3. RANI MANUVAL
4. MACHA SQL
5. MACHI WEB
6. KAJA PYTHON
Rollback: It acts as an undo function whenever we use rollback it all way to goes to previous Commit command
Syntax for Rollback command:
SQL>Rollback;
If we want to go to savepoint then we have to use
Syntax for Rollback command to go to savepoint:
SQL>Rollback to (savepoint_Name);
NOTE: Deleted records we can bring them back by using rollback but Truncated values we can’t bring back
/* deleting 4 th record*/
SQL> delete clg where sno=4;

FLASHBACK: After deleting a table that table will save in recyclebin. To bring back that deleted table we have to use this command
Syntax for Flashback command:
SQL> Flashback table table_name to before drop;
/* deleting clg table*/
SQL> drop table clg;
Table dropped.
/* retrieving the clg table*/
SQL> flashback table clg to before drop;
Flashback complete.
SQL> select * from clg;
SNO SNAME COURSE
---------- ---------- ---------
1. AJAY SQL
2. RAJA JAVA
3. RANI MANUVAL
PURGE: Whenever we want to delete a table from recyclebin then we have to use this command
Syntax for Purge command:
SQL>purge table table_name;
/* deleting clg table*/
